# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

BAArduinoController is a ROS 2 (Humble) Python package that bridges MoveIt2 trajectory commands to a physical 6-DOF robot arm (the "Bracket Arm") via serial communication to an Arduino with a PCA9685 servo driver. The codebase is written in German (comments, UI labels, variable names).

## Architecture

The system has three tiers:

1. **ROS2-PC**: Runs MoveIt2/RViz for motion planning, sends `FollowJointTrajectory` actions over DDS (WiFi)
2. **Raspberry Pi**: Runs this package — `Ros2Bridge` (action server) + `robot_state_publisher`, translates ROS trajectories to serial binary packets
3. **Arduino**: Receives binary servo commands over USB serial, drives PCA9685 via I2C

Key modules:
- `BAArduinoController/Ros2Bridge.py` — ROS 2 node with two `FollowJointTrajectory` action servers: `/arm_controller/follow_joint_trajectory` (joints 0-4) and `/gripper_controller/follow_joint_trajectory` (joint 5). Converts radians to servo degrees using per-joint offsets.
- `BAArduinoController/Sender.py` — Serial protocol layer. Sends binary packets (start marker `0xAA`, 6 angles as uint8, duration as uint16 LE, XOR checksum). Flush marker `0xFF` stops movement. Serial config from `.env` file.
- `GUIController.py` — Standalone tkinter GUI for direct servo control (no ROS required). Reads `config.json` for servo labels/offsets.
- `urdf/bracket_arm.urdf` — Robot description (6 joints: joint_0 through joint_5)
- `launch/pi_bridge.launch.py` — Launches `robot_state_publisher` + `Ros2Bridge` on the Pi

## Build & Run Commands

### ROS 2 package (Raspberry Pi)
```bash
# Activate ROS environment
micromamba activate ros_env

# Build (--symlink-install required for .env discovery, and avoids rebuilds for Python changes)
cd ~/ros2_ws
colcon build --packages-select BAArduinoController --symlink-install

# Source and launch
source ~/ros2_ws/install/setup.bash
ros2 launch BAArduinoController pi_bridge.launch.py
```

### GUI Controller (standalone, no ROS)
```bash
source .venv/bin/activate   # or venv/bin/activate on Pi
pip install -r requirements.txt
python GUIController.py
```

### Serial port configuration
Copy `.env.example` to `.env` and set `SERIAL_PORT` (e.g. `/dev/ttyACM0`) and `BAUD_RATE`. The port can change between reboots (`ttyACM0` → `ttyACM1`); check with `ls /dev/ttyACM*`.

**Important:** The Pi package must be built with `--symlink-install` so that `Sender.py` can find the `.env` via `os.path.realpath(__file__)` resolving back to the source tree.

## Serial Protocol

Binary packet format: `[0xAA] [angle0..angle5 as uint8] [duration_ms as uint16 LE] [XOR checksum]` — 10 bytes total. Arduino responds with free buffer slot count. `0xFF` flushes the Arduino movement buffer (emergency stop).

## Joint Mapping

6 servo joints with these rest offsets (degrees): Base=90, Schulter=0, Ellenbogen=0, Hand Gelenk=180, Hand Dreh=90, Greifer=45. The `map_ros_to_servo` function adds the ROS radian value (converted to degrees) to these offsets, clamping to 0-180.

## Related Repository: MoveIt Config

The MoveIt2 configuration lives in a separate repo at `../ba_arduino_controller_moveit_config/`. It is a CMake package (`ba_arduino_controller_moveit_config`) generated by the MoveIt Setup Assistant.

### MoveIt Planning Groups (SRDF)
- **arm**: kinematic chain `base_link` → `tcp_link` (joint_0 through joint_4), KDL solver
- **gripper**: joint_5 only (with `joint_5_mimic` as passive joint)
- **arm_with_gripper**: full chain `base_link` → `gripper_finger_left_link`
- Named pose **"home"**: joint_0=1.5708, joint_1=0, joint_2=0, joint_3=3.14, joint_4=1.5708, joint_5=0.76 (radians)

### MoveIt Controllers
Two `FollowJointTrajectory` controllers matching the action servers in `Ros2Bridge.py`:
- `arm_controller` → joints 0-4 → action topic `/arm_controller/follow_joint_trajectory`
- `gripper_controller` → joint 5 → action topic `/gripper_controller/follow_joint_trajectory`

### Key MoveIt Config Settings
- `trajectory_execution.allowed_start_tolerance: 0.0` — start tolerance check disabled because servos have no encoders (reported position from Ros2Bridge may differ from physical)
- `default_velocity_scaling_factor: 0.1` / `default_acceleration_scaling_factor: 0.1` — conservative defaults
- All joints: max_velocity 1.0 rad/s, no acceleration limits

### Build & Launch (ROS2-PC)
```bash
cd ~/ros2_ws
colcon build --packages-select ba_arduino_controller_moveit_config
source ~/ros2_ws/install/setup.bash

# Real robot (MoveGroup + RViz, connects to Pi via DDS)
ros2 launch ba_arduino_controller_moveit_config robot.launch.py

# Demo/simulation (MoveGroup + RViz + ros2_control FakeSystem, no Pi needed)
ros2 launch ba_arduino_controller_moveit_config demo.launch.py
```

## ROS 2 Dependencies

`rclpy`, `control_msgs`, `action_msgs`, `sensor_msgs`, `robot_state_publisher`, `joint_state_publisher`. DDS: CycloneDDS (`RMW_IMPLEMENTATION=rmw_cyclonedds_cpp`), `ROS_DOMAIN_ID=1` for PC-Pi communication.
